# Incident Detection, Triage & Response

This project demonstrates the core incident response lifecycle within a Security Operations Center (SOC) environment. Using case management, threat intelligence, digital forensics, and security monitoring tools, the project focuses on detecting security incidents, performing triage, preserving evidence, and coordinating response activities.

The objective of this project is to develop practical experience with alert management, incident investigation, threat validation, evidence collection, and incident response workflows through hands-on exercises and documented assessments.

## Overview

Incident detection and response are critical functions within modern SOC operations. In this project, multiple security platforms were used to investigate alerts, validate indicators of compromise (IOCs), preserve forensic evidence, and document response actions.

Activities included alert classification, incident ticket creation, threat intelligence validation, evidence collection, forensic preservation, escalation procedures, incident reporting, and response coordination. The project demonstrates how security analysts progress from initial alert detection to containment, remediation, and stakeholder communication.

## Tools & Environment

**Wazuh** — Used for security monitoring, alert generation, and incident investigation.

**TheHive** — Used for case management, incident tracking, and escalation workflows.

**CrowdSec** — Used for threat blocking and containment actions.

**Velociraptor** — Used for evidence collection, memory acquisition, and forensic analysis.

**VirusTotal** — Used for IOC validation and reputation analysis.

**AlienVault OTX** — Used for threat intelligence enrichment and indicator validation.

**Metasploitable-2** — Vulnerable target system used for attack simulation and response testing.

**Virtual Lab Environment** — Provides an isolated environment for safe security testing and incident response exercises.
