# Threat Intelligence, SOAR & Automation

This project demonstrates the integration of threat intelligence, security orchestration, automated response, and advanced incident handling within a Security Operations Center (SOC) environment. Using SIEM, SOAR, threat intelligence platforms, and digital forensics tools, the project focuses on enriching security alerts, automating response actions, and improving incident response efficiency.

The objective of this project is to develop practical experience with threat intelligence integration, incident escalation, automated workflows, threat hunting, evidence preservation, and security orchestration through hands-on exercises and documented assessments.

## Overview

Modern SOC operations rely on threat intelligence and automation to reduce response times and improve detection accuracy. In this project, security events were enriched using external threat intelligence sources, investigated through structured triage workflows, and escalated using case management platforms.

Activities included log correlation, anomaly detection, IOC validation, threat hunting, incident escalation, evidence collection, automated playbook creation, and incident response orchestration. The project demonstrates how security teams can combine threat intelligence and automation to enhance operational efficiency and strengthen defensive capabilities.

## Tools & Environment

**Wazuh** — Used for security monitoring, alert generation, and threat detection.

**Splunk Phantom (SOAR)** — Used to automate incident response workflows and alert escalation.

**AlienVault OTX** — Used for threat intelligence enrichment and IOC reputation analysis.

**VirusTotal** — Used for threat intelligence validation and IOC investigation.

**TheHive** — Used for case management and incident escalation workflows.

**Velociraptor** — Used for forensic evidence collection and analysis.

**CrowdSec** — Used for threat containment and automated response actions.

**Metasploitable-2** — Vulnerable target system used for attack simulation and detection testing.

**Virtual Lab Environment** — Provides an isolated environment for safe security testing and automation exercises.
